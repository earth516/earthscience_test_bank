# Apps Script 수정 안내 (개념 확인하기 + 기록 기능용) 개념 확인하기는 새 시트 **Concepts**를 읽고, 학생 기록을 **ConceptResponses**에 남깁니다. 그래서 Apps Script를 아래처럼 고치고 **새 버전으로 재배포**해야 합니다. (4곳) --- ## 1. `loginAndGetData` 분기에 concepts 추가 ```javascript } else if (action === 'loginAndGetData') { const loginResult = checkLogin(e.parameter.studentId); if (loginResult.status !== 'ok') { return jsonResponse(loginResult); } const questionsResult = getQuestions(); const conceptsResult = getConcepts(); // ← 추가 return jsonResponse({ status: 'ok', 학번: loginResult.학번, 이름: loginResult.이름, questions: questionsResult.data, concepts: conceptsResult.data // ← 추가 }); ``` ## 2. `doGet` 분기에 saveConcept(+선택 getConcepts) 추가 ```javascript } else if (action === 'getConcepts') { return jsonResponse(getConcepts()); } else if (action === 'saveConcept') { return jsonResponse(saveConcept({ 학번: e.parameter.학번, 이름: e.parameter.이름, 개념ID: e.parameter.개념ID, 제목: e.parameter.제목, 학생답안: e.parameter.학생답안, 점수: e.parameter.점수 })); ``` ## 3. 함수 두 개 새로 추가 (`getQuestions` 아래 아무 곳) ```javascript function getConcepts() { const sheet = SpreadsheetApp.openById(SHEET_ID).getSheetByName('Concepts'); if (!sheet) return { status: 'ok', data: [] }; // 시트 없으면 빈 목록 const data = sheet.getDataRange().getValues(); if (data.length < 2) return { status: 'ok', data: [] }; const headers = data[0]; const rows = []; for (let i = 1; i < data.length; i++) { const row = {}; headers.forEach((h, idx) => { row[h] = data[i][idx]; }); rows.push(row); } return { status: 'ok', data: rows }; } function saveConcept(data) { const ss = SpreadsheetApp.openById(SHEET_ID); let sheet = ss.getSheetByName('ConceptResponses'); if (!sheet) { // 시트 없으면 자동 생성 + 헤더 sheet = ss.insertSheet('ConceptResponses'); sheet.appendRow(['타임스탬프', '학번', '이름', '개념ID', '제목', '학생답안', '점수']); } sheet.appendRow([ new Date(), data.학번, data.이름, data.개념ID, data.제목, data.학생답안, data.점수 ]); return { status: 'ok', message: '개념 기록 저장 완료' }; } ``` ## 4. 반드시 재배포 배포 → 배포 관리 → ✏️ → **버전: 새 버전** → 배포. (URL 유지) 점검: 브라우저에서 `[API_URL]?action=loginAndGetData&studentId=20101` → 응답에 `"concepts":[...]`가 보이면 성공. --- ## Concepts 시트 만들기 시트 이름: **Concepts** (대소문자 정확히) | 열 | 헤더 | 설명 | 예시 | |---|---|---|---| | A | 개념ID | **기록 구분용 이름표**(자유). 없어도 동작하지만, 기록에서 어느 지문인지 알아보려면 적어두길 권장 | `atm-c-01` | | B | 대단원 | breadcrumb 표시 | 대기 | | C | 소단원 | 목록 그룹 | 대류권 | | D | 제목 | 지문 카드 제목 | 대류권의 특징 | | E | 지문 | 본문. **빈칸은 `[[정답]]`으로 감싸기** | 아래 예시 | **지문(E열) 예시:** ``` 대류권은 높이 올라갈수록 [[기온]]이 낮아진다. 공기의 [[대류]] 현상이 활발해 기상 현상이 나타난다. 성층권에는 [[오존/오존층]]이 있어 자외선을 흡수한다. ``` - `[[ ]]`로 감싼 부분이 전부 빈칸(랜덤 없음). - 한 빈칸에 여러 정답 허용: 슬래시 → `[[오존/오존층]]` - 셀 안 줄바꿈(Alt+Enter) 반영됨. 채점은 공백/대소문자 무시. > **개념ID란?** 코드가 채점·표시에 쓰는 값이 아니라, 선생님이 "몇 번 지문"인지 알아보고 **기록 시트에서 지문을 구분**하기 위한 이름표입니다. 비워도 앱은 동작하지만, 기록을 남기기로 했으니 적어두는 걸 추천합니다. --- ## ConceptResponses 시트 (기록) - **따로 안 만들어도 됨** — 학생이 처음 채점하는 순간 Apps Script가 시트를 자동으로 만들고 헤더도 넣습니다. - 미리 만들고 싶으면 시트 이름 `ConceptResponses`, 헤더: | A | B | C | D | E | F | G | |---|---|---|---|---|---|---| | 타임스탬프 | 학번 | 이름 | 개념ID | 제목 | 학생답안 | 점수 | - **한 지문 = 한 줄**로 기록됩니다. - 학생답안 예시: `기온(O) | 바람(X) | 오존(O)` — 각 빈칸의 입력값과 정오(O/X)를 한 칸에 모음. - 점수 예시: `2/3`
